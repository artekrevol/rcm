Step 4 — Update the unit calculator to use location-based rates
Find the rate lookup in the claim wizard — currently it queries hcpcs_rates for a hardcoded VA rate. Replace this with a location-aware lookup.
New rate query — add this to the existing GET /api/billing/hcpcs/search endpoint response, and also expose it as a separate endpoint for the wizard to call when a code is selected:
sql-- GET /api/billing/va-rate?code=G0299&location=AUSTIN
SELECT 
  vlr.facility_rate as rate_per_unit,
  vlr.location_name,
  hc.unit_type,
  hc.unit_interval_minutes,
  hc.description_plain
FROM va_location_rates vlr
LEFT JOIN hcpcs_codes hc ON hc.code = vlr.hcpcs_code
WHERE vlr.hcpcs_code = $1
AND vlr.location_name = $2
AND vlr.is_non_reimbursable = false
LIMIT 1
If no location is configured in practice settings, fall back to a national average:
sqlSELECT ROUND(AVG(facility_rate)::numeric, 2) as rate_per_unit
FROM va_location_rates
WHERE hcpcs_code = $1
AND is_non_reimbursable = false
