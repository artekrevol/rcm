## ITEM 4 — ERA Auto-Posting Rules Per Payer

Add ERA auto-posting configuration to the existing Payer Settings.

**In the payer edit dialog, add a new "ERA Settings" section with these toggles:**

All toggles are OFF by default except where noted.

- Auto-post clean ERAs — OFF by default. When ON: ERAs with no errors, no denials, and no $0 allowed lines post automatically without biller review
- Auto-post secondary payments — ON by default
- Auto-post contractual adjustments (CO-45, CO-42, CO-97) — OFF by default. When ON: these adjustment codes are automatically written off without biller review
- Auto-post refunds and reversals — ON by default
- Hold for review if payment does not match expected amount — ON by default

**Behavior when "Auto-post clean ERAs" is ON for a payer:**
When an 835 ERA file is received from that payer and it meets all enabled auto-post conditions, the ERA is posted automatically:
- Payment amounts are applied to the matching claims
- CO- adjustments flagged for auto-post are written off
- ERA status is set to "Auto-Posted"
- A system note is added to each affected claim: "Payment auto-posted from ERA [check number] on [date]"

When any condition is NOT met (e.g., a denial exists in the ERA), the ERA falls through to the manual review queue regardless of the auto-post setting.

**For the demo org:**
Pre-configure the VA Community Care payer with:
- Auto-post clean ERAs: ON
- Auto-post contractual adjustments (CO-45): ON
- All others: default

This reflects the real-world VA billing workflow where CO-45 (charges exceed fee schedule) is always a contractual write-off and requires no biller decision.

## ITEM 5 — Denial Recovery Agent

This is the feature that differentiates ClaimShield from competitors. Competitors translate denial messages. ClaimShield should diagnose and guide the fix.

**Where it lives:**
On the claim detail page, when a claim has a denial or rejection status, show a new "Recovery" panel below the existing denial explanation card.

**The Recovery panel contains:**

**Step 1 — Root Cause (already partially done by AI explanation — extend it)**
Below the existing AI explanation, add a "Root Cause" line that categorizes the denial into one of these buckets based on the CARC code:
- Missing Information (CO-16, CO-4) → "Required field is missing or invalid"
- Authorization (CO-15, CO-57) → "Prior authorization issue"
- Timely Filing (CO-29) → "Claim filed outside the timely filing window"
- Duplicate (CO-18) → "Duplicate claim submission"
- Eligibility (CO-27, CO-26) → "Patient eligibility issue"
- Coding (CO-4, CO-50, CO-97) → "Procedure or diagnosis code issue"
- Contract / Fee Schedule (CO-45) → "Contractual adjustment — write off"

**Step 2 — Recommended Action**
Based on the root cause bucket, show a specific recommended action:
- Missing Information → "Open claim and complete: [list the specific fields that map to this CARC code]"
- Authorization → "Verify prior authorization number and resubmit"
- Timely Filing → "Generate Proof of Timely Filing Letter" (button that triggers the existing letter generator)
- Duplicate → "Check if claim was already paid under a different claim number"
- Eligibility → "Run eligibility check for this patient" (button that triggers eligibility lookup if built, otherwise shows instructions)
- Coding → "Review procedure and diagnosis code combination"
- Contract → "Apply CO-45 write-off and close claim" (button that applies the adjustment)

**Step 3 — Fix It Button**
For denials where the fix is a form field correction (Missing Information, Authorization, Coding):
Show a "Fix This Claim" button that:
1. Opens the claim wizard in edit mode
2. Scrolls directly to the section containing the field(s) that need to be corrected (use the CARC-to-field mapping below)
3. Highlights the specific field(s) with a yellow border and a tooltip: "This field may have caused the denial"

**CARC to field mapping for Fix It routing:**
- CO-16 → Additional Info tab, Claim Note field
- CO-4 → Service lines, CPT/modifier fields
- CO-15 → Primary Insurance, Authorization # field
- CO-57 → Primary Insurance, Authorization # field
- CO-50 → Service lines, CPT code field
- CO-97 → Service lines, CPT + modifier combination
- CO-29 → Additional Info tab, Delay Reason Code field

**Step 4 — One-Click Resubmit**
After the biller makes corrections, show a "Validate & Resubmit" button at the bottom of the Recovery panel that:
1. Runs the existing pre-submission validation (same as the validate-edi endpoint)
2. If validation passes → triggers Office Ally SFTP submission
3. If validation fails → shows the validation checklist with specific failures highlighted
4. On successful resubmission → marks the original denial as "Fixed - Resubmitted" in the Claim Tracker
