## ITEM 2 — Clinic Home Page (Clinic Admin View)

Add a "My Practice" page for org-level admins (Daniela and future clinic admins). This gives them a single command center for their clinic.

**Route:** /billing/clinic
**Access:** admin role only (within their own org)
**Sidebar:** Add "My Practice" as a nav item between Dashboard and Claims in the billing sidebar

**Section 1 — Practice Profile**

Display (not a form — read-only with an Edit button that links to /billing/settings):
- Practice name
- Primary NPI
- Tax ID
- Billing address (street, city, state, zip)
- Phone number
- Default Place of Service

Clearinghouse Status badge:
- If Stedi API key is configured: green badge "✓ Connected to Stedi"
- If Office Ally SFTP is configured: green badge "✓ Connected to Office Ally"  
- If neither: red badge "⚠ Clearinghouse not configured" with a link to Settings → Clearinghouse

**Section 2 — Team**

Table of all users in this organization:
- Columns: Name, Email, Role (color-coded badge), Last Active, Status
- Role badge colors: admin = blue, rcm_manager = green, intake = teal
- Last Active: pull from the most recent session or activity_log entry for that user. Show "Never" if no activity.
- Status: Active (green) or Inactive (grey)
- Actions per row: Edit Role (opens a simple dialog to change role), Deactivate/Reactivate toggle

"Invite New User" button at top right of the section — opens the existing user invite dialog (same as /billing/settings/users). Do not rebuild the invite flow — reuse what exists.

**Section 3 — Setup Health**

Permanently visible version of the onboarding checklist (not dismissible). Same 6-step logic as the existing onboarding card:
1. Practice information configured
2. At least one provider added
3. At least one payer added
4. Clearinghouse connected
5. Claim defaults saved
6. First claim created

Each step shows:
- ✅ green check + step description if complete
- 🔴 red gap + step description + a "Fix This" link to the relevant settings tab if incomplete

Title: "Clinic Setup Health" — makes it clear this is an ongoing health check, not a one-time checklist.

**Section 4 — Quick Stats (last 30 days)**

Four stat cards in a row:
- Claims submitted
- Claims paid
- Active follow-ups (claims with a follow_up_date set and status not paid)
- Open denials (claims with status = denied and no mark-fixed event)

Pull from existing data — no new tables needed.
