# ClaimShield — Required/Optional Chip Extension (Sprint 1e Follow-up)

## Context

Sprint 1e shipped Required/Optional chips on First Name, Middle Name, and Last Name only. The original spec called for chips across all user-facing forms. Tuesday's demo needs the chips to extend through the claim wizard (where Rachel actually got stuck on Friday — fields like DX Pointer, Modifier, and Delay Reason Code) and the rest of the patient form. The chip component is already implemented — this prompt just extends the existing pattern to the remaining fields.

## Scope rules

- Use the existing chip component implementation from First/Last Name. Do not create a new component or new styles.
- Only extend coverage to: (1) all remaining patient create + edit form fields, (2) all claim wizard fields across every step. Skip payer setup, provider setup, and user management — those are out of scope for Tuesday and can wait.
- Static rules only. Do not add conditional logic ("this field becomes required when X"). Chips reflect the static requirement.
- Validation behavior unchanged. Chips are visual only.
- If a field name in the tables below doesn't match the actual form field name in code, use the closest equivalent. Do not invent new fields or skip fields that exist in code but aren't listed.

## Section 2 — Claim wizard: all steps

Apply chips to every input across every step of the claim wizard. Step through Patient → Service Lines → Diagnoses → Provider → Review and chip every field.

**Required:**
- Rendering Provider
- Service Start Date
- CPT/HCPCS Code
- Hours (or per-unit count, whichever the payer's rate mode shows)
- Rate (Hourly Rate or Rate per Unit, depending on payer mode)
- ICD Diagnosis Pointer
- Primary ICD-10 Diagnosis Code
- Authorization Number / Referral Code
- Place of Service

**Optional:**
- Service End Date
- Modifier
- Secondary ICD-10 Diagnosis Code
- Delay Reason Code
- Replacement / Void / Cancel flags
- Ordering Provider *(defaults to Rendering Provider when blank)*
- Notes / Comments

For any claim wizard field that exists in code but isn't in either list above, default to Optional unless validation already marks it required.
