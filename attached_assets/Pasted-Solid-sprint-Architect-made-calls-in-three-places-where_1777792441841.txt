**Done well**

The four-state machine (`drafting → submitted → in_review → {approved, rejected}`) is cleaner than my prompt's six-state version. Combining `submitted_for_qa` and `running` into `submitted`, and `revision_required` into `rejected`, removed two states whose distinction was semantic noise. The `in_review` state when checks complete and a human is needed is exactly the right beat.

`HardFailBlockedError → 409 hard_fail_blocked` instead of letting it surface as a 500 is a real-world fix. Sprint 4's override modal needs to handle this status code specifically; a 500 would have made that flow harder to write. The architect saw this two sprints ahead.

The `qa.override.attempt` audit log row written *before* `NotImplementedError` throws is exactly the right pattern for a stubbed action. Even though the override doesn't work in Sprint 3, every attempt is captured. When Sprint 4 ships the real override flow, the audit trail is continuous — no gap. This is the kind of detail that matters six months from now when someone asks "did anyone try to override during Sprint 3?"

The "crashed check → run status `error`, no silent pass-through; hard-fail still wins over crash" rule is correct and non-obvious. If Originality.ai times out, the run is `error`, not `passed`. If brand-voice crashes but originality fails hard, the run is `failed`, not `error`. Future Decision Engine code (Sprint 7) will need to weigh these states; getting the precedence right now means less rework later.

The "writers stay in ContentForge with a SubmittedForReview panel that polls /review/:id every 4s" pattern is better than my prompt's redirect-to-status-view approach. Writers don't context-switch out of their tool. The 4s poll is reasonable — Realtime would be better but adds Supabase Realtime complexity to ContentForge for one panel; polling is simpler and good enough.

The narrow self-fetch exception on `/review/:id` (writers can poll their own submission but not others) is the correct authorization shape. Reviewer-only would have broken the writer's submitted view; fully open would have leaked drafts cross-pod.
