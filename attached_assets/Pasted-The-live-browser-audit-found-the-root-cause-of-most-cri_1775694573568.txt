Every HCPCS code search returns {"error":"relation \"va_location_rates\" does not exist"} which causes the React white screen crash.   created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(hcpcs_code, location_name, carrier, locality_code)
);

FIX 2 — Patient dropdown click-to-select (Critical)
The patient search dropdown in claim wizard Step 1 closes without registering the selection when clicked. The root cause is the input's onBlur handler firing before the onClick on the dropdown item completes.
Find the patient search dropdown component in the claim wizard Step 1. On every dropdown result item, change onClick to onMouseDown and add e.preventDefault():
typescript// Find the dropdown item that selects a patient — change this:
onClick={() => handleSelectPatient(patient)}

// To this:
onMouseDown={(e) => {
  e.preventDefault(); // prevents blur from firing before selection
  handleSelectPatient(patient);
}}
After fixing, verify: type a partial name, results appear, click a result — patient name shows in the selection area and Next button enables. Test with a normal click speed, not a slow deliberate click.
