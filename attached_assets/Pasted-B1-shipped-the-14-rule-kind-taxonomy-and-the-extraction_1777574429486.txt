T2 — Active evaluator: referrals
Add a new evaluator function in rules-engine.ts:
typescriptasync function evaluateReferralsRules(
  claim: Claim,
  patient: Patient,
  referralRules: ExtractionItem[]
): Promise<RuleViolation[]>
Logic:

If referralRules is empty, return empty array (no rules in corpus for this context).
For each referral rule, parse extracted_json into the ReferralsResult shape (from B1).
If referral_required = true for the patient's plan_product:

If patient.pcp_referral_number is empty: emit a block violation with severity = 'block', rule_kind = 'referrals', liability = liability_if_missing (from rule), and source_quote from the rule.
If patient.pcp_referral_number is present but patient.pcp_id is empty: emit a warn violation noting "Referral number recorded but no PCP on file — verify before submission."


For exception-list cases (e.g., exceptions: ["self-referral_for_OB"]), check the claim's CPT codes against the exception conditions and skip the block if they match.

Each violation carries: ruleId (the extraction item ID), sourceQuote, sourceDocumentId, reviewedBy, lastVerifiedAt. These display in the claim wizard's "View source" dialog.
T3 — Active evaluator: documentation_timeframe
Similar pattern for documentation_timeframe rules:
typescriptasync function evaluateDocumentationTimeframeRules(
  claim: Claim,
  docTimeframeRules: ExtractionItem[]
): Promise<RuleViolation[]>
Logic:

Only fires for claims where status IN ('pending_appeal', 'under_audit', 'records_requested').
For each rule, calculate expiry_date = trigger_event_date + days_allowed.
If today > expiry_date: emit block — record window has lapsed.
If today is within 48 hours of expiry_date: emit warn with hours remaining.
Otherwise: emit informational (visible in the claim's history, not blocking).

The trigger_event_date is the date the records request was issued, the appeal was filed, or the audit was initiated. For the initial build, take this from a new records_request_date column on claims (add via migration if not present).
