## Step 4: Update the HCPCS search endpoint

The current search endpoint at `GET /api/billing/hcpcs/search` only queries the `hcpcs_codes` table. Update it to search across all three tables in a single UNION ALL query so CPT codes from physician referrals return real descriptions instead of triggering the manual fallback.

Find the existing search endpoint in routes.ts and replace the SQL query with:

```sql
SELECT 
  code,
  description_official as description,
  description_plain,
  unit_type,
  unit_interval_minutes,
  default_pos,
  requires_modifier,
  notes,
  'hcpcs' as source,
  (SELECT rate_per_unit FROM hcpcs_rates 
   WHERE hcpcs_code = hcpcs_codes.code 
   AND payer_name = 'VA Community Care' 
   AND end_date IS NULL 
   LIMIT 1) as va_rate
FROM hcpcs_codes
WHERE is_active = true
AND (
  code ILIKE $1
  OR description_official ILIKE $1
  OR description_plain ILIKE $1
)

UNION ALL

SELECT 
  code,
  description as description,
  NULL as description_plain,
  unit_type,
  NULL as unit_interval_minutes,
  NULL as default_pos,
  false as requires_modifier,
  NULL as notes,
  'cpt' as source,
  NULL as va_rate
FROM cpt_codes
WHERE (
  code ILIKE $1
  OR description ILIKE $1
)
AND NOT EXISTS (
  SELECT 1 FROM hcpcs_codes WHERE code = cpt_codes.code
)

ORDER BY
  CASE WHEN code ILIKE $2 THEN 0 ELSE 1 END,
  source,
  code
LIMIT 20
```

Parameters: `$1` = `%${query}%` (for ILIKE contains search), `$2` = exact code match for ranking.

The `NOT EXISTS` clause on the CPT union prevents duplicate results when a code appears in both tables (some codes are in both HCPCS and CPT).

**Also update the ICD-10 search in the claim wizard.** Find where ICD-10 codes are currently searched (likely a small hardcoded array of 20 common codes) and replace it with a database query:

```typescript
// New ICD-10 search endpoint
// GET /api/billing/icd10/search?q=hypertension

app.get('/api/billing/icd10/search',
  requireRole('admin', 'rcm_manager'),
  async (req, res) => {
    const q = String(req.query.q || '').trim();
    if (!q || q.length < 2) return res.json([]);

    const results = await pool.query(`
      SELECT code, description, is_header
      FROM icd10_codes
      WHERE is_active = true
      AND is_header = false
      AND (
        code ILIKE $1
        OR description ILIKE $2
      )
      ORDER BY
        CASE WHEN code ILIKE $1 THEN 0 ELSE 1 END,
        LENGTH(code),
        code
      LIMIT 15
    `, [`${q}%`, `%${q}%`]);

    res.json(results.rows);
  }
);
```

Update the ICD-10 search field in the claim wizard to call this endpoint instead of filtering the hardcoded array. Keep the hardcoded array as a fallback if the endpoint fails.
