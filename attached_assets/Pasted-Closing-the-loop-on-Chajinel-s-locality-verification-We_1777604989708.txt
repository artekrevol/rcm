WHAT TO DO:

Step 1 — Download the CMS ZIP-to-Carrier-Locality crosswalk file

The file is published on CMS's Fee Schedule General Information 
page:
https://www.cms.gov/medicare/payment/fee-schedules/general-information

Look for "Zip Code to Carrier Locality File" — published 
quarterly. Get the most recent version (likely titled something 
like "ZIP5_CY2026.txt" or "ZIPCODE_CARRIER_LOCALITY_Qx_2026.zip").

If that page doesn't have the file directly, alternative source:
https://www.cms.gov/Medicare/Medicare-Fee-for-Service-Payment/FeeScheduleGenInfo/index.html

ResDAC (a CMS data repository) also documents this file:
https://resdac.org/articles/medicare-zip-code-carrier-locality-file

The file format is typically fixed-width or pipe-delimited text 
with columns: ZIP, state, carrier, locality, rural indicator, 
plus-4 indicator, year, etc.

Step 2 — Ingest into the schema

Use the cms_zip_locality table from the Reimbursement Reference 
Data sprint:

CREATE TABLE IF NOT EXISTS cms_zip_locality (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  zip_code TEXT NOT NULL,
  zip_plus_4 TEXT,
  state TEXT NOT NULL,
  mac_carrier TEXT NOT NULL,
  locality_code TEXT NOT NULL,
  rural_indicator TEXT,
  effective_date DATE NOT NULL,
  termination_date DATE,
  source_url TEXT,
  ingested_at TIMESTAMP DEFAULT NOW(),
  UNIQUE (zip_code, zip_plus_4, effective_date)
);

Should populate with ~40,000+ rows covering all U.S. ZIPs.
