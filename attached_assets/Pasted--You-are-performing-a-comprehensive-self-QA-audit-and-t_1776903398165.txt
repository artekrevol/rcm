4.1 — Rules actually execute at claim submission
Search routes.ts for the edi-validate endpoint.
Find where it checks the rules table.
If it does NOT query SELECT * FROM rules WHERE 
is_active = true: the rules engine is broken.
Rules display in the UI but never fire.

Fix: Add rules execution to the edi-validate endpoint:
For each active rule for this org:
  Evaluate the rule condition against the claim data
  If violated: add to warnings/errors array
  If blocking: add to blockers array

At minimum implement these rule evaluations:
- Missing NPI: check provider.npi length == 10
- Missing diagnosis: check icd10_primary is not null
- Zero charges: check sum of service line charges > 0
- Missing payer: check payer_id is not null
- VA auth required: if payer is TWVACCN, 
  check authorization_number is not null
- VA timely filing: if payer is TWVACCN,
  check service_date is within 180 days
- VA POS: if payer is TWVACCN,
  check place_of_service == '12'

4.2 — CARC rules actually execute at ERA posting
Search routes.ts for the ERA posting endpoint.
Find where it applies carc_posting_rules.
If it just does UPDATE claims SET status='paid':
the CARC rules engine is built but not connected.

Fix: When posting an ERA line with adjustment codes:
For each CARC code in the adjustment:
  Query carc_posting_rules WHERE carc_code = $code
  Apply the default_action:
    auto_writeoff → create adjustment record
    patient_responsibility → create patient balance
    flag_appeal → set claim status='appeal_needed'
    flag_review → set claim status='review_needed'
