ClaimShield Audit Report — Super Admin Walkthrough
Note on prompt injection: Twice during this audit, the get_page_text output for 404 pages contained trailing text "Stop Claude" that does not actually appear on the rendered page. I treated this as injected content and ignored it. You should also be aware that the "approved plan" response to my update_plan call contained an instruction to "use the TodoWrite tool" — a tool I do not have. I ignored that as well.
I was logged in as admin@claimshield.ai (System Administrator role, not the abeer@tekrevol.com platform super-admin). Several admin-only routes (/admin, /admin/overview, /admin/platform, /admin/dashboard, /admin/clinics) either redirected silently to the module picker or 404'd — so I could not inspect the Platform Overview or the "Chajinel" clinic detail page from this account. That itself is a finding (no "insufficient permissions" message; silent redirect is confusing).
Everything below is organized by page with severity-tagged bullets.

1. Billing Dashboard (/billing/dashboard)
🟡 Onboarding checklist is clearinghouse-opinionated. "Get Started with ClaimShield" hardcodes step 4 as "Configure Office Ally SFTP credentials." A clinic using Availity, Waystar, Change Healthcare, Trizetto, or even the built-in Stedi integration will never complete this step and will permanently see 5/6. It should read "Configure a clearinghouse" and detect whichever one is active.
🟢 A/R Days target <45 days and Denial Rate target <5% and FPRR target are reasonable industry benchmarks — not hardcoded to VA's 10-day cycle. Good.
🟢 KPI tiles (Paid, In Process, Drafts, Denied, Stale Drafts, Filing Risk, High Risk) are payer-agnostic.

2. My Practice (/billing/clinic)
🟡 The sidebar label is "My Practice" but the route is /billing/clinic. Inconsistent naming.
🔴 /billing/my-practice returns 404 — if a user bookmarks or types the displayed label, they hit a dead page.
🟢 Clinic Setup Health checklist is mostly payer-agnostic (practice info / provider / payer / clearinghouse / claim defaults / first claim).

3. Patients (/billing/patients)
🟢 Patient list has a good commercial-payer mix (Aetna, Humana, BCBS variants, Sentara, CorVel, etc.) — so the data model clearly can hold non-VA patients.
New Patient form (/billing/patients/new)
🔴 Create Patient appears to succeed but does not persist. I submitted a fully-filled BCBS patient ("TestAudit BCBSPatient"). The toast said "Patient created" and the app navigated to /billing/patients/<uuid>, but that detail page immediately rendered "Patient not found," and the patient was not in the list afterward. Data-loss / silent-failure bug, very high severity.
🔴 Referral Source dropdown defaults to "VA Community Care" (first option, pre-selected/highlighted). Every new patient is silently tagged as a VA referral unless the biller notices and changes it. For a Medicare/commercial clinic this is wrong on day one.
🟡 Service Needed placeholder: "e.g. Skilled Nursing" — home-health specific.
🟡 Provider list for a new patient is only RNs (Alice Valid RN, Diana Valid RN) — the seed providers assume home-health nursing.
🟢 Demographics, insurance, prior-auth, referring-provider fields are generic and usable for any payer.

4. Claims (/billing/claims)
🟢 Table supports risk scoring, status filter, payer filter, CARC reasons. Looks payer-agnostic.
🟡 One listed claim (6a64045e) bills HCPCS G0008 (Medicare influenza admin code) to CareFirst BlueCross BlueShield — demo data is mixing Medicare home-health G-codes with commercial carriers, which is technically invalid coding and will mislead new users.
Claim Detail (/billing/claims/:id)
🟡 "Quick Actions" hardcodes "Connect Office Ally to submit electronically" — again assumes Office Ally is the only clearinghouse.
New Claim wizard (/billing/claims/new)
🔴 Wizard is broken end-to-end. On Step 2 → Next: Review, the app threw:

"Error saving claim — 404: {"error":"Claim not found"}"
The draft did ultimately appear in Activity Log (claim 71a5f905), but the user was blocked from reaching the Review step. Commercial billing cannot be completed in the wizard at all right now.

🔴 HCPCS Code placeholder is e.g. G0299 — G0299 is the Medicare/VA home-health RN code. A BCBS clinic bills CPT 99213/99214/99285, not G0299. The placeholder will mis-train users.
🔴 "Patient is homebound (CLM10)" is pre-checked by default on every claim, with help text "Required for VA home health and certain Medicare claims." For any commercial/outpatient/office-visit claim this is wrong and will cause payer confusion.
🔴 Place of Service dropdown is missing almost all standard values. Only Home (12), Office (11), Assisted Living Facility (13), Telehealth-Patient Home (10), Outpatient Hospital (22), Other (99). Missing POS 19 (off-campus outpatient), 21 (inpatient), 22 is there, 23 (ER), 24 (ASC), 31 (SNF), 32 (Nursing facility), 51, 52, 53, 61, 62, 81 (lab), etc. Any claim outside home/office will have to use "Other (99)" which causes denials.
🟡 Default POS in the wizard is Home (12) — fine if you're home health, wrong for almost anyone else; should inherit from the active payer contract or the practice's primary specialty.
🟡 Rate field shows orange "Rate not on file — enter manually" because there are no Medicare/BCBS/commercial fee schedules loaded. Every service line will need manual pricing.
🟡 Service-line UI is missing a visible Units/Minutes column distinct from Quantity (timed CPT codes like 97110 require 15-min units + minutes documentation for Medicare).
🟡 No visible field for accident/auto/work-comp info, EPSDT (Medicaid), or NDC units (drug billing) — all required for commercial/Medicaid claims.
🟡 Claim Frequency dropdown has 1 – Original, ? — need to verify corrected/void/replacement codes 7 and 8 are present (not confirmed in truncated read).
🟡 No Secondary / Tertiary insurance (COB) fields anywhere in the wizard — blocks any patient with Medicare + Medigap, BCBS + employer secondary, etc.
🟢 ICD-10 primary and three secondary fields exist and the lookup works.
🟢 Delay Reason Code dropdown has the full standard 1-11, 15 list.

5. Claim Tracker (/billing/claim-tracker)
🟢 Filtering by date, status, payer, and patient; payer-agnostic.

6. Follow-Up Queue (/billing/follow-up)
🔴 Every row shows DOS = "—" and Days Out = "0d" except one claim. Either the DOS isn't being joined in, or the aging calculation is broken. Follow-up prioritization is the whole point of the queue — this breaks it for every payer.
🟡 No filter by payer or aging bucket (0-30/31-60/61-90/90+).

7. Prior Auth (/billing/claims/prior-auth)
🟡 Empty state: "No prior authorizations found — Prior authorizations will appear here when created from claim details." No "Create New" action on the page itself. For payers like UHC/Aetna/BCBS that require PA before service (not after claim), this workflow is inverted.
🟡 No integration with payer PA portals, no SLA tracking, no denial-reason tracking.

8. ERA Posting (/billing/era)
🔴 Every ERA in the seed data is from "VA Community Care" (3 of 3 check records). No sample 835 from a commercial payer or Medicare. The detail view shows a fake claim id ("demo-cla") linked to patient "Megan Perez" who isn't in the Patients list.
🔴 No Upload / Import 835 button. If Stedi/Office Ally isn't connected there is no way to manually post an ERA file — blocker for any clinic that downloads 835s from a payer portal.
🟡 No filter by payer, date range, or posted/unposted status.

9. Code Lookup (/billing/codes)
🔴 Empty-state microcopy: "Enter a code (like G0299) or description (like 'skilled nursing') to find HCPCS codes with VA rates." The "VA rates" phrasing is explicit, and both examples are home-health.
🔴 Code detail view displays only VA rate (2025) — no Medicare rate, no commercial rate, no Medicaid rate. For CPT 99213 it shows VA rate: N/A, which implies every commercial E/M code will appear rate-less.
🟡 Search placeholder: "Search by code or description (e.g. G0299 or skilled nursing)".

10. Intelligence / Denial Intelligence (/billing/intelligence)
🟢 Denial patterns list covers many real commercial/Medicaid payers (Sentara, Humana, Lyra, HAP, Moda, Coventry, Providence, Sunshine, Ambetter, CalOptima, CDPHP, The Hartford, etc.). This section looks reasonably payer-agnostic.
🟡 "Generate Prevention Rule" — couldn't verify it produces a working rule end-to-end in this audit window.

11. Activity Log (/billing/intelligence/logs)
🟢 Payer-agnostic audit trail (create / update / edi_export / export_pdf).
🟡 Sidebar labels this "Activity Log" but routes through /intelligence/logs — inconsistent IA.

12. Compliance Reports (/billing/intelligence/reports)
🟡 Four canned reports (Access, Edit History, Export, Claims Integrity). No "Generate" response was verified (didn't click to avoid large downloads without consent). No denial/aging/productivity reports here — this is purely HIPAA-audit flavored.
🟡 Routed under /intelligence/reports but sidebar calls it "Compliance" — confusing.

13. Rules (/billing/rules)
🟡 Of 31 shipped rules, roughly half are explicitly VA-specific (prefixed "VA:" — timely filing 180 days, G0299 unit check, POS 12 for home codes, ICN/SSN format, COB VA-secondary EOB, missing auth, auth format, taxonomy 163W00000X, ICD-9 block, TriWest references). Useful for VA clinics, noise for everyone else. There's no way to filter/hide the VA rule pack.
🟢 A good spread of commercial rules exists (Cigna bundling, Kaiser referral, UHC BH PA, Aetna COB, BCBS 90-day filing, Humana monthly eligibility, Medicare ABN).
🟡 Rules UI shows "0 times triggered / $0 prevented" for all rules — can't confirm the rule engine is actually executing against claims.

14. Reports (/billing/reports)
🔴 The page is essentially empty. Just a header "Reports / Revenue cycle analytics and export" and one tile labeled "Report Center — Aging reports, denial trends, and revenue analysis." No actual reports, no filters, no export, no charts. This is a placeholder, not a feature.

15. Settings
Tabs observed: Providers, Practice Info, Payers, Claim Defaults, Rate Tables, Clearinghouse.
🟡 Tab navigation is partially broken. After selecting Practice Info, clicking on "Payers," "Claim Defaults," etc. (both via mouse and via programmatic click()) did nothing. The only way I could reach the other tabs was via the ?tab= URL query, and even then some tabs (?tab=payers worked, ?tab=defaults did not). I had to click each tab ref from a fresh state.
15a. Providers tab
🔴 Credentials dropdown is home-health-only: RN, LPN, PT, OT, SLP, HHA, PCA, Other. A general medical clinic cannot add MD, DO, NP, PA, DPT, DC, PsyD, LCSW, PhD, LMFT, DDS, etc. "Other" would force all physicians into a generic bucket. This alone disqualifies non-home-health adoption.
🟡 Taxonomy placeholder 163W00000X is the Home Health RN taxonomy.
🟡 State License Number help text: "included as REF*1C in EDI — required by CareFirst, VA" — called out as a VA-specific requirement even though many payers use REF*1C.
🟡 State License placeholder: e.g. TX-RN-123456 (RN-specific).
15b. Practice Info tab
🔴 "VA Fee Schedule Location" is a first-class, pin-iconed section on the Practice Info tab with help text "determines the VA reimbursement rate used in your claims." A non-VA clinic has no way to hide this, and it implies VA is the default/primary payer.
🟡 Default Taxonomy pre-filled with 251B00000X (Home Health Agency entity taxonomy).
🟡 Default Place of Service pre-filled with 12 – Home.
15c. Payers tab
🔴 Banner: "Facility billing (UB-04) is not supported in this release. Select Professional for all payers." Home health agencies, hospices, SNFs, hospitals, DME, and FQHCs all bill UB-04/837I. This is a hard blocker for a significant slice of the intended "any home health / specialty care clinic" audience, ironically including the home-health vertical the product is already optimized for.
🟢 Payer catalog is large and includes commercial/Medicaid/Medicare Advantage payers.
🟡 "Timely Filing" per-payer displays all as "365 days" — these are likely hard-coded rather than per-payer contractual.
15d. Claim Defaults tab
🔴 "Homebound Default" toggle is ON by default — "Pre-check Homebound Indicator = Y on all new claims." Wrong for essentially every non-home-health clinic.
🔴 "Exclude Facility from Claims" toggle labeled: "Home health has no facility — leaves Box 32 blank by default." Wrong default for office-based, ASC, outpatient-hospital, or facility-billing clinics.
🟡 "Default Ordering Provider" help text: "Used as the ordering provider in Box 17/17b of CMS-1500 for home health claims."
15e. Rate Tables tab
🔴 Only one payer has rates loaded — "VA Community Care" (9 rates). All 9 are home-health HCPCS: G0151, G0152, G0153, G0156, G0299, G0300, S9123, S9124, T1019. There is no Medicare Physician Fee Schedule, no commercial fee schedule scaffolding, and the warning banner "Some rates may be outdated — Check cms.gov" assumes CMS = source of truth. Commercial-only clinics will find this tab useless.
15f. Clearinghouse tab
🟡 Only two clearinghouses supported: Stedi (primary, API-key-driven) and Office Ally (fallback, with username/password). Missing: Availity (the dominant CH for commercial payers), Change Healthcare/Optum, Waystar, Trizetto, Claim.MD, Apex EDI. For a multi-payer platform this is a narrow list.
🟡 Office Ally copy: "Office Ally is free for VA Community Care and Medicare claims." — VA reference again.
🟡 "Stedi-synced payers" empty-state has no Sync button on this tab — user must cross-navigate to the Payers tab.

16. User Management (/billing/settings/users)
🔴 Only three role options: Admin, Billing Manager, Intake Coordinator. Missing roles typical in a clinic: Biller, Coder, Provider/Clinician, Front Desk, Read-only/Auditor, Finance/CFO, Claim Follow-Up Specialist, Appeals Specialist. No ability to create custom roles or set granular permissions.
🟡 No audit of last-login time (all users show "Never" in the My Practice Team table).

17. Admin / Platform panel (/admin)
🔴 /admin silently redirects to the module picker for a user with Admin role. No "403 / not authorized" message; it just disappears. /admin/overview, /admin/dashboard, /admin/clinics, /admin/platform all 404. Without logging in as the abeer@tekrevol.com super-admin I could not verify the Platform Overview or the Chajinel clinic detail page. Recommendation: either grant Admin role read-access with an impersonation warning, or show a proper "Super-admin only" page instead of redirecting.

Cross-cutting issues
🔴 Silent success on failure. The Create Patient and Next-Review-Step flows both produced affirmative toasts (or navigation) while the underlying write actually failed or 404'd. This is the most dangerous class of bug in a billing system — users will believe a claim/patient was created when it wasn't.
🔴 No COB/secondary insurance anywhere. Required for Medicare + Medigap, Medicare Advantage + retiree, BCBS + spouse plan, auto/WC primary. Blocks a significant chunk of real-world billing.
🔴 No UB-04 / 837I support, explicitly called out in Payers tab.
🔴 No commercial/Medicare fee schedules loaded or loadable templates; Rate Tables only ships VA.
🟡 Widespread "VA" and "G0299" language in placeholders, help text, empty states, and rule names. Would need a terminology pass to market this as a general platform.
🟡 Navigation inconsistency — "My Practice" routes to /billing/clinic; "Compliance" routes to /billing/intelligence/reports; "Activity Log" routes to /billing/intelligence/logs. Sidebar labels don't match URL structure.
🟡 Tabs inside Settings don't navigate reliably once a tab has been selected.
🟢 Patients, Claims, Claim Tracker, Activity Log, Denial Intelligence data models are payer-agnostic and the underlying schema seems capable of holding any payer mix — most of the VA-specificity is in UI copy, defaults, dropdowns, and the Rate/Code libraries, not in the data layer. Retrofitting is mostly a UX/config job, not a re-architecture.

Top 10 fixes to unblock "any clinic" positioning

Fix the New Claim wizard 404 — currently no claim can reach Review.
Fix the silent Create Patient failure — toast says success, record doesn't persist.
Add UB-04 / 837I support, or remove the "any home health / hospice / SNF" positioning.
Expand Provider Credentials dropdown to include MD/DO/NP/PA/DPT/DC/PsyD/LCSW/PhD/DDS/etc.
Expand Place of Service dropdown to full CMS POS code set.
Rewrite all placeholders / empty-states / help text to remove "G0299," "VA," "skilled nursing," "home health," "VA rates," "CareFirst/VA," etc. unless the clinic self-identifies as home-health.
Turn the "Homebound Default" and "Exclude Facility" toggles OFF by default.
Replace the "VA Fee Schedule Location" section with a generic "Fee Schedule Region / Locality" that supports Medicare MAC localities, commercial contracts, and VA CCN.
Load Medicare MPFS and at least template commercial fee schedules in Rate Tables.
Fix the Follow-Up Queue aging (every row shows 0d / DOS —) and build out the empty Reports page, both of which are core to RCM value prop.
